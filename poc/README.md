# PoC（使い捨て技術スパイク）

特定の**技術課題が成立するか**を確かめる使い捨てスパイクの作業場。**どのフェーズでも使う**（特に P4 着手前の de-risk、P5 後の機能追加での実現性テスト）。

- **問いは「技術的にできるか」**。製品・コンテンツの検証（＝MVP / `mvp/`）とは別。1 スパイク = 1 つの技術的問い。
- **`app/` を汚さないための隔離箱。** スパイクコードは本実装に持ち込まない（知見をもとに `app/` で書き直す）。
- コードは使い捨て。**結論・学びは `docs/research/`（トピックノート）か該当 spec の `validation.md` に promote** する。
- 残す価値のない試行は commit せず捨ててよい。残すなら `poc/<topic>/` に分け、不要になったら削除。
- commit prefix は `[poc]`。
