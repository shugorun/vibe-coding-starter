# MVP Technical（使い捨ての技術構成）

## 記録方法

- MVP を**最小コストで動かすために選んだ技術だけ**を書く。**設計は書かない**（本実装の設計・スタックは P3 `app-design/technical.md` → `04-ARCHITECTURE.md`）。
- これは**使い捨て**。本実装(P5)には引き継がない（本実装スタックは P3 で 0 から決める）。
- 実験で切り替える変種（プロンプト / スキーマ / 検索設定）は `switches.md`。ここは土台の構成だけ。

## 構成

| 項目 | 採用 | メモ |
|---|---|---|
| アプリ種別 | 未設定 | 本実装と別でよい（例: 本番モバイルでも MVP はウェブ） |
| 言語 / ランタイム | 未設定 | |
| 主要フレームワーク / ライブラリ | 未設定 | |
| 起動 / 動かし方 | 未設定 | |

## 選定理由

- 最小コストで中核体験を検証できるか、を基準に選ぶ（1-2 行）。
