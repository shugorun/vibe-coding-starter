# MVP Design Index

P2（MVP）の作業場。`mvp/` の使い捨てプロトタイプで**何を試し・何を学んだか**を記録する。**生ログはここ**に置き、確定した学び・コンテンツは `docs/app-design/`（P3）/ `docs/progress/` に昇格する（二重持ちしない）。P2 が終われば記録として残る（research と同じ扱い）。

## ファイル

| ファイル | 役割 |
|---|---|
| [scope.md](scope.md) | 作る MVP のスコープ（可変） |
| [experiments.md](experiments.md) | 実験ログ（変種・設定・結果・学び／時系列） |
| [switches.md](switches.md) | 切り替え管理（生きている変種と切替方法＝可逆性の台帳） |
| [feedback.md](feedback.md) | ユーザ（あなた）のフィードバック |
| [problems.md](problems.md) | 感じている問題（確定した不具合は `docs/issues/` へ昇格） |

## 現在地

| 項目 | 内容 |

## 運用

- 実験変種（プロンプト / スキーマ / 検索設定など）は**切替可能に保ち、動いていた状態を壊さない**。変種は `switches.md`、結果は `experiments.md` に記録（`mvp/README.md`）。
- 確定した学び・コンテンツは `docs/app-design/` / `docs/progress/` に昇格する。
