プロダクトの進め方と優先順位。

## 優先順位ルール

1. ユーザ価値の中核に近いものを先にやる。
2. 技術リスクが高いものは PoC を早める。
3. 仕様が揺れているものは実装前にレビューゲートを置く。
4. 個人開発で運用負荷が高いものは後回しにする。

## 更新ルール

- 新しい要件や spec が増えたらバックログに反映する。
- マイルストン完了時は `05-PROGRESS.md` と progress に記録する。
- 優先順位の判断が非自明なら ADR に残す。
