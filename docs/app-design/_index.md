# App Design Index

作ると決めた 1 つのアイデアを具体化する場所。ideas は「何を作るか」、app-design は「それをどういうアプリにするか」を扱う。

ここは **考え中の作業場**。固まった内容は `01-PRODUCT.md`（プロダクト）/ `04-ARCHITECTURE.md`（技術）に反映し、本フォルダからはリンクする。両者を二重管理しない。

## ファイル

| ファイル | 役割 | 更新タイミング |
|---|---|---|
| [overview.md](overview.md) | アプリ概要、対象ユーザ、価値、スコープ | 作るアイデアを決めたとき / 方向性が変わったとき |
| [mechanics.md](mechanics.md) | アプリの仕組み、主要フロー、ドメイン構造 | 体験やロジックの仕組みを考えたとき |
| [frontend.md](frontend.md) | 画面、UX、ナビゲーション、UI 方針 | フロントの構成や見た目を考えたとき |
| [technical.md](technical.md) | 技術方針、構成、API、データ、運用 | 技術選定や実装方式を考えたとき |

## 状態

|---|---|
| 元 idea | 未設定 |
| 現在の設計ステータス | todo |

## 運用

- 1 つの採用アイデアに対する設計だけを書く。
- 別アプリに乗り換える場合は、既存内容を必要に応じて退避してから更新する。
- 各ファイル冒頭の「記録方法」に従って追記する。
- 大きくなりすぎたら、まず章を整理する。分割は 300 行を超えそうな場合だけ検討する。
