# App Design Templates

```markdown
# App Overview

## 記録方法
- アプリの方向性が変わったら更新する。
- 古い方向性は「変更履歴」に日時付きで残す。

## status
draft

## 元 idea
- docs/ideas/...

## 一言説明

## 対象ユーザ

## 解決する課題

## 提供価値

## MVP スコープ

## 非スコープ

## 成功条件

## 未確定事項

## 変更履歴
### YYYY-MM-DD HH:mm
- 変更前:
- 変更後:
- 理由:
```

## technical.md

```markdown
# Technical Design

## 記録方法
- 技術選定、構成、API、DB、認証、運用リスクを書く。
- 非自明な決定は ADR に分離する。

## プラットフォーム & 言語
app-design に入ったら最初に決める（アプリ種別 → 言語 → スタック）。一方通行決定なのでユーザ確認を取る。
| 項目 | 決定 | 候補・トレードオフ | status |
|---|---|---|---|
| アプリ種別 | | Web / モバイル / デスクトップ / CLI・ツール / その他 | todo |
| 言語 | | 種別に応じて | todo |
| ランタイム / 主要フレームワーク | | 言語決定後に絞る | todo |

## 技術スタック候補

## アーキテクチャ

## データ設計

## API / 外部連携

## 認証 / 権限

## テスト / 検証

## 運用 / 監視

## 技術リスク

## 未確定事項

## 更新ログ
```
