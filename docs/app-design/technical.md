# Technical Design

## 記録方法

- 技術選定、構成、API、DB、認証、運用リスクを書く。ここは検討中の作業場。
- 確定した技術構成は `04-ARCHITECTURE.md` に反映する。
- 非自明な決定は ADR に分離し、このファイルからリンクする。
- 調査が必要なものは research にトピック単位で記録する。

## プラットフォーム & 言語（本実装。P3 で 0 から決める）

本実装(P5)のアプリ種別・言語・主要フレームワークを **ここ(P3) で 0 から決める**。MVP(P2) のプロトタイプ技術は**引き継がない**（例: モバイル製品を MVP では軽くウェブで試した、でも本実装はモバイルで決め直してよい）。引き継ぐのはコンテンツの学びだけ。これは一方通行決定なので、選択肢とトレードオフを提示してユーザ確認を取り、`04-ARCHITECTURE.md` のスタック表と `01-PRODUCT.md`「対象プラットフォーム」に pin する。
