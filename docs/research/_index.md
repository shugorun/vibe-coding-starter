# Research Index

技術調査、競合調査、参考資料の記録。日時単位ではなくトピック単位で 1 ファイルにまとめる。

## ステータス

- `todo` - 調査予定
- `doing` - 調査中
- `done` - 結論あり
- `stale` - 古くなった可能性あり

## 一覧

| トピック | ファイル | status | 最終調査日時 | 現在の結論 | 関連 |
|---|---|---|---|---|---|
| なし | | | | | |

## テンプレート

新しい research は [_TEMPLATE.md](_TEMPLATE.md) をコピーして作成する。

## 運用

- ファイル名は `<topic-slug>.md`。
- 同じトピックの調査は同じファイルに追記する。
- 各調査エントリには必ず日時を書く。
- 何を検索したか、何が分かったか、判断への影響を分けて書く。
- 情報が古くなりそうなものは再調査トリガを明記する。
