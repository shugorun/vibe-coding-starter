# Ideas Index

何を作るかのアイデア壁打ち。1 アイデア = 1 ファイル。

近いアイデアが変化した場合は新規ファイルを増やさず、同じファイルの「現在のアイデア」を更新し、更新前の内容を「過去のアイデア履歴」に日時付きで残す。

## ステータス

- `inbox` - 未整理
- `exploring` - 壁打ち中
- `candidate` - 作る候補
- `promoted` - requirement に昇格
- `discarded` - 採用しない

## 一覧

| ファイル | 現在のアイデア | status | 最終更新 | 調査トピック | 次アクション |
|---|---|---|---|---|---|
| なし | | | | | |

## テンプレート

新しい idea は [_TEMPLATE.md](_TEMPLATE.md) をコピーして作成する。

## 運用

- ファイル名は `<idea-slug>.md`。日時はファイル名に入れず、ファイル内部の「作成日時」「最終更新」「追加情報ログ」に記録する。
- 1 ファイルに 1 つのアイデア系列を記録する。
- 追加調査・追加情報は同じ idea ファイルの「追加情報ログ」に追記する。
- ほぼ同じアイデアの変化は「現在のアイデア」を更新し、旧内容を「過去のアイデア履歴」に移す。
- 別物のアイデアになった場合だけ新規ファイルに分ける。
