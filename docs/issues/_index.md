# Issues Index

## 定義

**issue = このプロジェクトの解決すべき問題**（不具合・ブロッカー・想定外挙動）で、原因や対処の文脈を残す価値があるもの。コード / 設定 / 仕様を直せば消える性質のもの。

- **起票する**: その場で直しきれず残る / 後で直す・判断が要る / 診断が非自明で残す価値がある。
- **起票しない**: その場で直って終わる自明なもの（直した事実は commit が持つ）。
- **environment との違い**: マシン・OS・シェル・ツールの性質で、直しても消えない制約は issue ではなく `docs/09-ENVIRONMENT.md`。迷ったら「コード / 設定を直せば消えるか?」→ Yes は issue。
- **未整理の困りごととの違い**: MVP 中の感覚的な困りごとは `docs/mvp-design/problems.md`（低い敷居）。解決対象として確定したら issue へ昇格。

## ステータス

- `Open` - 未解決
- `Resolved` - 解決済。`resolved/` 配下に移動

## 起票テンプレ

```markdown
# Issue NNNN: <タイトル>

## ステータス
Open

## 起票日
YYYY-MM-DD

## カテゴリ
build / runtime / dependency / spec / api / ux / product / etc.

## 影響範囲
- ...

## 現象
何が起きているか。

## 再現手順（あれば）
1. ...

## 試した対処
- ...

## 解決手段
Resolved 時に追記。

## 関連
- ADR:
- spec:
- progress:
```

## 運用

- 解決すべき問題に当たったら即起票する（その場で直って終わる自明なものは除く）。
- 解決時は `open/` から `resolved/` に移動し、index を更新する。
- 再発時は新規 issue を起票し、過去 issue を参照する。
