# Work

自然文の作業依頼を受けたら、このフローで進める。`/catchup-project`（開始）と `/wrapup-project`（終了）の間の通常作業の本体。

**固定の手順を一直線に踏むのではない。** 前後に固定点（着手前の確認・区切りの commit）があり、その間は作業ループ。規則を読む・罠→issue・判断→ADR は順番ではなく**作業中つねに守る不変条件**。これで起票漏れとユーザ側タスク放置による失敗を防ぐ。

```text
確認（着手前・未解決なら着手しない）
 └ 依頼を作業単位（= コミット1個分）に分解
    └ 各単位: 作業（コード ⇄ docs、規則を読む / 罠→issue / 判断→ADR）→ /checkpoint
```

## 0. 現在地を把握

`docs/05-PROGRESS.md` で現在フェーズを確認する。コーディング先はフェーズで変わる: **P2 は `mvp/`（使い捨て）、P5 は `app/`（本実装・0 から）。** フェーズ移行が絡むなら `docs/03-ROADMAP.md` のゲートを確認し、未チェックなら移行しない。例: **P2(MVP) に入るには P1 ゲートで「MVP を作る手段（種別・言語）」が決まっていること**（最小コスト優先・本実装と別でよい）。本実装(P5)の種別・言語・スタックは P3 で 0 から決め、MVP の技術選定は引き継がない。

## 1. 確認（着手前のゲート・固定点）

ゴール達成までに**先に潰すべき不確実性**を洗い出す。未解決のまま着手しない（保留して進めると失敗する）。

- ブロッカーになる質問（仕様・一方通行決定）。一方通行決定は `docs/02-GUIDELINES.md`「レビューゲート」に従い確認。
- **ユーザがやるべきタスク**: アカウント作成、API キー / 認証情報、課金、外部サービス設定、実機・端末準備、ユーザしか決められない選択。
- 引き返せる範囲は確認不要 —— 判断して進め、根拠は progress に残す。
- 確認は一度きりではない。作業中に新しいブロッカーが出たら、その時点でここに戻る。

## 2. 分解

依頼を**作業単位（= コミット1個分の動く増分）**に割る。大きい依頼ほど複数単位になる。各単位を順に下の「作業ループ」で進め、単位ごとに checkpoint する（1 単位 = 1 commit = 1 progress エントリ）。

## 3. 作業ループ（各単位）

単位を仕上げる。中身はコード / docs / 調査 / 混在のどれでもよく、コードと docs を行き来してよい。**作業中つねに守る不変条件**:

- **触る前に規則を読む**: コードは `docs/08-CODING.md` + `docs/09-ENVIRONMENT.md`、docs は `docs/02-GUIDELINES.md`「作業中の記録ルール」。新しい言語の初回は formatter / linter / typecheck を入れる（`08-CODING.md`）。
- **スタックに従う**: `docs/04-ARCHITECTURE.md` の表（pin 後は ADR 先行、探索/MVP 期は progress 一行 + `technical.md`）。
- **罠を踏んだら → その場で issue 起票**（`docs/issues/open/`）。環境起因なら `docs/09-ENVIRONMENT.md`「既知の罠」にも 1 行。
- **非自明な判断をしたら → ADR 起票**（`docs/decisions/`）。MVP 期の小さな選定は progress「決めたこと」一行でよい（2層: `02-GUIDELINES.md`「意思決定の記録」）。
- 実装で仕様が変わったら spec / app-design を直す。

## 4. checkpoint（単位の区切り・固定点）

`/checkpoint` を行う（progress 1 エントリ → 検証緑 → commit。push しない）。手順は checkpoint スキル。次の単位があれば「3. 作業ループ」に戻る。

## 厳守事項

- 一直線に踏まない。**着手前の確認**と**区切りの commit** が固定点、規則読み・罠→issue・判断→ADR は随時発火する不変条件。
- **issue / ADR の起票を後回しにしない**（起票漏れがこのプロジェクトの主要な失敗）。
- ユーザ側タスクを保留したまま実装を進めない。
- フェーズ移行ゲートが未チェックなら次フェーズに進まない。
- 軽微な 1 単位（1 行修正等）でも、規則確認と「記録 + commit」は省かない。
