# Catchup Project

**発動条件**: description のとおり。
**成功基準**: 追加質問なしに現在地・次の行動・レビュー待ち・未解決 issue を4行で報告できる。
**書き込みなし**: このスキルは読むだけ。記録は /work と /checkpoint が担当する。

## 手順（読書量の目安 ≤150行。ユーザに「どれを見るか」は聞かない）

1. 宣言してから `git status --short` と `git log --oneline -5`。未コミット変更があればそれを現在地として認識する（今回の依頼と同件なら続行、別件なら先に /checkpoint で確定）。
2. `docs/STATE.md` — 全節を読む（節構成はファイル自身が正）。
3. `docs/ROADMAP.md` — 現フェーズのゲートの節だけ読む。
4. `docs/DECISIONS.md` — 末尾 10 行だけ読む。
5. `docs/issues/` — frontmatter `status: open` のファイル名だけ拾う。
6. 実装依頼があるときだけ追加で: `docs/ARCHITECTURE.md`（スタック表・コード判断規則）と `docs/ENVIRONMENT.md`。

## 報告形式（途中報告せず、最後に1回だけ）

```text
[現在地] <フェーズ / 今やっていること（未コミットがあれば併記）>
[次の行動] <STATE の「次」、またはユーザ依頼への次アクション>
[レビュー待ち] <件数と対象 / なし>
[未解決] <open issue 件数>
```

ユーザが具体的な依頼を同時に出していれば、サマリの後そのまま /work で進める。

## 厳守事項

- STATE と実態（git・実ファイル）が食い違ったら実態を正とし、食い違いを /checkpoint の STATE 上書きで直す。
- 手順 6 以外で実装ファイルやコード規則を読み込まない（読書量の予算を守る）。
