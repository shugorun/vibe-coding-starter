# Grill

出典: [mattpocock/skills](https://github.com/mattpocock/skills) の grill-me / grill-with-docs を統合・改変（MIT — 表記は [LICENSE](../../../LICENSE)）。

**発動条件**: description のとおり。
**成功基準**: 次の作業単位の着手前提がすべて確認され、確定した内容が下記の表の家に書き込まれている。

## 進め方

計画の全側面について、共通理解に達するまで質問し続ける。決定ツリーの各分岐を順に下り、決定間の依存関係を1つずつ解消する。

- 質問は1つずつ。各質問に推奨回答を添える。
- コードベースや docs の探索で答えられる質問は、探索してから聞く（ユーザに聞かない）。
- ユーザの発言が既存の docs・コードと食い違ったら、その場で指摘してどちらが正かを確定する（例: 「用語節では X はこう定義されているが、今の話は Y に聞こえる — どちらか?」）。
- 曖昧語・多義語には正準語を提案して1語に決める。
- 概念間の関係・境界の議論には、エッジケースを突く具体的なシナリオをぶつけて精密化する。

## 書き込み（確定した瞬間に行う。バッチでためない）

| 確定したもの | 書き込み先 |
|---|---|
| 用語（正準語と1-2文の定義。避ける言い方も併記） | [docs/PRODUCT.md](../../../docs/PRODUCT.md)「用語」節 |
| プロダクト事実（課題・対象ユーザ・非目標・制約） | docs/PRODUCT.md の該当節 |
| 技術構成（スタック・依存方向・コード判断規則） | [docs/ARCHITECTURE.md](../../../docs/ARCHITECTURE.md)（スタック表の pin・逸脱はユーザ承認後） |
| 一方通行決定（基準・書式: [docs/adr/README.md](../../../docs/adr/README.md)） | docs/adr/ に1件1ファイル（ユーザ確認後） |

- 用語は「それが何であるか」を定義する。「何をするか」の実装詳細は書かない。
- adr は安易に作らない。「戻すのが高コスト」「文脈なしには驚く」「実際のトレードオフの結果」の3つが揃うときだけ提案する。1つでも欠けるなら見送る（可逆な決定は CLAUDE.md の常設ルールのとおり DECISIONS.md に1行）。

## 厳守事項

- セッション途中でも、確定した内容はその場で書き込む（途中で切れても成果が残る）。
- このスキルの書き込み先は上の表の4箇所のみ。コードは書かない。
