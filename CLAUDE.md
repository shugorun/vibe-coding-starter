# CLAUDE.md

このリポジトリは **docs 駆動の個人開発プロジェクト**。コードと並行して、判断・要件・仕様・詰まり・次タスクを `docs/` に残しながら進める。

## このプロジェクトでの最優先ルール

**`docs/` 配下の md を能動的に作成・更新することは、このプロジェクトで明示的に要求された動作。**「ドキュメントを勝手に作らない」という通常のデフォルトはここでは適用しない。ただし無制限に書くのではなく、下記の「記録の判断基準」に従う。

## 記録の判断基準（量ではなく重要度）

記録するのは「将来の自分がこれ無しだと **間違える / 迷う**」情報だけ。

記録する:

- 方向転換、確定した判断、ユーザが明示した好み・制約
- コードや git から再導出できない背景・理由
- 後で見返す必要がある未確定事項

記録しない:

- やったこと自体（git が記録している）
- 既存の記述の言い換え
- まだ揺れている思考の途中経過

## 書く場所（追記ログ と curated doc を分ける）

- **追記専用ログ** = `docs/progress/`、idea の「追加情報ログ」、research の「調査ログ」。気軽に頻繁に追記してよい。時系列なので散らからない。
- **現在の真実を表す curated doc** = `docs/01-PRODUCT.md` / `docs/04-ARCHITECTURE.md` / `docs/app-design/` / `docs/requirements/` / `docs/specs/`。マイルストン級の変化があったときだけ編集し、常にタイトに保つ。

細かい話はログ側へ流し、curated doc は滅多に触らない。これで読み返しやすさを保つ。

## 書くタイミング（cadence）

- 作業の流れの途中で自動的に書き込まない。**話題の区切り・タスク切替・`/wrapup-project` でまとめて**書く。
- 記録すべきか迷ったら、doc を組み替えずに `docs/progress/` に1行メモを残す。
- 大きな記録が必要なときは、勝手に書かず「ここまでを記録しておく？」と1行で提案する。

## セッション運用

- 開始: 文脈を戻したいときは `/catchup-project`。
- 通常作業: チャット指示を読み、`docs/02-GUIDELINES.md` に従う。
- 終了: `/wrapup-project`。
- `reviewed` status はユーザの明示承認があるものにだけ付ける。

## 入口

- `docs/00-INDEX.md` — docs 全体の索引と読む順
- `docs/05-PROGRESS.md` — 現在地・次タスク・レビュー待ち
- `docs/02-GUIDELINES.md` — 運用ルールの詳細
