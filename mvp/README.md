# MVP（使い捨て）

フェーズ **P2** の作業場。**最小フロント＋中核機能だけ**を素早く作り、このアプリの中核体験（コンテンツ）が何かを確定するための使い捨てプロトタイプ。

- コードの綺麗さ・設計は問わない。速く動かして学ぶことが目的。
- **作り方（種別・言語 / FW）は最小コスト優先で選ぶ。本実装の種別とは別でよい**（例: 本番モバイルでも MVP はウェブで試す）。選んだ手段はここに 1 行記録してよいが、本実装には引き継がない。
- **本実装(P5)はここを参照せず `app/` で 0 から書く（コードも技術選定も）。** ここで適当に書いたコード/ロジックは本実装に持ち込まない。
- 確定したコンテンツと学び（活かす / 捨てる）は `docs/progress/` と `docs/app-design/` に残す。
- P2 のゲート（→P3）は `docs/03-ROADMAP.md`。
